每一层做了什么 & 为什么这样设计

Mel 频谱输入

原始 1.2s 音频 → 短时傅里叶变换 → 128 个 Mel 频带 × 54 时间帧。Mel 尺度模拟人耳听觉，压缩高频维度。

频率维 128 足够覆盖车载告警（300-8000Hz），时间维 54 帧给 GRU 足够步长

Conv1-3（频率池化）

3 层卷积，每层后接 2× 频率下采样。通道数 1→32→64→128。只压频率维，保留时间维。

频率维从 128 → 64 → 32 → 16，时间维始终保留 → 给 GRU 提供完整的时序序列

Conv4（无池化）

128 ch，3×3 卷积，无池化。保持 128×16×54 输出。

最后一层卷积精炼特征，不再降维

AdaptiveAvgPool2d((1,None))

频率维全局平均池化到 1。输出 128×1×54 → Reshape 为 54×128。

将每个时间帧的频域信息压缩为一个 128 维特征向量

双向 GRU ×2（核心）

输入 54 步 × 128 维。128 隐藏单元，2 层，双向 → 输出 256 维/步。Dropout 0.3。

前向 GRU 学"响→停"关系，后向 GRU 学"停→响"关系。双向拼接 = 完整上下文。2 层 = 更深层次时序表征

MaxPool over Time

对 GRU 输出的 54 个时间步取最大值。54×256 → 256。

取最大激活值，确保只要有任一帧触发，分类头就能响应。比取平均更适合稀疏事件检测

分类头

256 → 64 → 8（Dropout 0.3）。8 个 Sigmoid 独立输出。

独立 Sigmoid = 多标签：窗口内可以同时输出 ACC=1, FCW=1

为什么不用 Transformer

自注意力在 54 帧短序列上收益小，数据量不够时容易过拟合。GRU 逐步记忆恰好匹配"先响再停"的局部节奏。
